MAD Roadmap (2016+)

[Disclamer: this roadmap includes only the development of the MAD 'next-generation',and the schedule is strongly subjected to my availability on this project. The development process is complex and does not involve the MAD-team for the time being (only partial time of LD), but this might change starting from beta releases 0.1 (Fall 2016).]

Beta releases planning for 2016-2017:

The beta releases planning is an attempt to group and summarise the many detailed items described in the remaing part of this page. The serious development started in May 2016 (see contribution graph) with the integration of the code developped for the demonstrator, but the progress depends on my daily avalaibility (see punch card). Some items hereafter (ports and benchmarks) have as objectives to facilitate the transition from MAD-X to MAD 'next-generation', including backward propagation of improvements to MAD-X. We will deliver beta releases after each minor number in the releases repository.

  • 0.0.1: integration, help & testing framework (may-july).
  • 0.0.2: math, complex and linear algebra (august).
  • 0.0.3: beams, elements, sequences/lines, tables (september).
  • 0.0.4: survey tracking and plotting framework (october).

in //, port of LHC, HL-LHC, FCC-ee, FCC-hh sequences to new MAD.

  • 0.1.0: particle tracking without errors.
  • 0.1.1: close orbit search.

in //, unit tests to benchmark of all elements without errors.

  • 0.2.0: gtpsa (real and complex) finalised, integrated and tested.
  • 0.2.1: high order tracking without errors.
  • 0.2.2: linear normal forms and twiss without error.
  • 0.2.3: close orbit search revisited (alternate using TPSA).

in //, unit tests to benchmark high orders for all elements without errors.

  • 0.3.0: support for assemblies and errors.
  • 0.3.1: patches for displacement, assemblies and errors.
  • 0.3.2: particle tracking with errors.
  • 0.3.3: high order tracking with errors.

in //, unit tests to benchmark of all elements with errors all orders.

  • 0.4.0: optimisation framework with equality constraints (matching).
  • 0.4.1: optimisation framework with inequality constraints.

in //, unit tests to benchmark the matching capabilities with the existing.

  • 0.5.0: high order normal forms.
  • 0.5.1: extraction of optics functions from high order maps.

in //, unit tests to benchmark the results with the new book of E. Forest.

  • 1.0.0: first non-beta releases (end 2017).
